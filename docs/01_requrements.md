# プロジェクト名: EduBoard (エデュボード)
# 要件定義書 兼 設計書 (初期構築用)

## 1. プロジェクトの目的と概要
* **目的:** 中高一貫校受験に向けた1年間の学習スケジュールと単元リストを管理し、紙に印刷してアナログで運用するための可視化ツールを作成する。
* **利用環境:** Google Apps Script (GAS) の「Webアプリ」としてデプロイ。自分のみがアクセスできるセキュアな状態で運用する。
* **運用フロー:**
  1. スプレッドシートにスケジュールと単元（マスターデータ）を記載する。
  2. GASのWebアプリURLにアクセスし、最新データをブラウザで表示する。
  3. 画面上に表示された印刷最適化フォーマットをA4用紙に印刷し、学習時のチェックや誤答分析を手書きで書き込む。

## 2. 技術スタックとアーキテクチャ
ビルドツール（Webpack等）は使用せず、CDN経由でモダンなフロントエンドライブラリを積極的に活用し、グラフィカルかつ拡張性の高いSPA（Single Page Application）風のUIを構築する。

* **バックエンド (GAS / コード.gs):**
  * `SpreadsheetApp` を用いてシートのデータを取得し、フロントエンドで扱いやすい JSON 配列に変換して返すAPIの役割に徹する。
* **フロントエンド (HTML / JS):**
  * **フレームワーク:** Vue.js (CDN版) または Alpine.js（データとDOMのバインディングを容易にするため）。
  * **スタイリング:** Tailwind CSS (CDN版) を利用し、モダンでクリーンなUIを構築する。
  * **可視化ライブラリ:** 要件に応じて、CDNで導入可能な軽量ライブラリを適宜選定して利用すること。
  * **CSS要件:** 画面上のモダンな見た目とは別に、必ず `@media print` を活用し、A4用紙サイズでの印刷に最適化すること（背景色の印刷対応、改ページ制御、不要なUI要素の非表示）。

## 3. データ構造（スプレッドシートの列定義）
1行目をヘッダーとし、以下のカラムを持つ想定。
* `対象月` (数値: 4〜3)
* `種別` (文字列: 学習、模試、イベント等)
* `科目` (文字列: 国語、算数、理科、社会、全科)
* `大項目` (文字列: 単元の大きな括り)
* `単元名` (文字列: 具体的な学習内容や模試名)
* `備考` (文字列: データ上の補足事項)

## 4. 画面・出力仕様（初期実装の可視化フォーマット）
フロントエンドのフレームワークを活用し、タブ切り替え等で以下のビューをシームレスに表示・印刷できるようにする。

### ビューA: 全体俯瞰マトリクス（年間マップ）
* **目的:** 1年間の全体像をひと目で把握する。
* **レイアウト:**
  * マトリクス表形式（縦軸: 科目、横軸: 対象月）。
  * セル内に該当する「単元名」を箇条書きで配置。手書きチェック用の四角いボックス（`[ ]`）を先頭に付与する。
  * **要望:** 単なるHTMLのテーブルではなく、CSSフレームワークを用いてモダンで視認性の高いグリッドデザインにすること。

### ビューB: 月別・単元チェックリスト（書き込み用）
* **目的:** 月ごとのToDoリスト兼、手書きでガッツリ書き込むための台紙。
* **レイアウト:**
  * 対象月ごとにセクションを区切り、リスト化。
  * **表示項目:** 科目、大項目、単元名、備考。
  * **手書き用スペース:** 各行の右側に「完了チェック」の空欄と、広めの「誤答メモ・分析用余白」の列を必ず確保すること。
  * 印刷時は月ごとに改ページされるよう制御（`page-break-before: always;` 等）を行うこと。

## 5. 開発の進め方と初回依頼事項
本ドキュメントをベースに、まずはプロトタイプを作成し、その後対話を通じて機能やUIを拡張していく。

**【今回の依頼事項】**
1. GASプロジェクト用の `コード.gs` と `index.html` の初期コードを出力してください。
2. `コード.gs` に、スプレッドシートのデータを読み取り JSON で返すロジックを実装してください（シート名などの変数は仮のものでOKです）。
3. `index.html` に、Tailwind CSS と Vue.js (または Alpine.js) のCDNをインポートし、取得したデータを元に「全体俯瞰マトリクス」と「月別チェックリスト」を描画・タブ切り替えできるUIを実装してください。
4. 印刷用の `@media print` の初期設定を含めてください。
5. 出力後、私がGAS環境で動作確認を行うためのセットアップ手順（スプレッドシートの用意からデプロイまで）を簡潔にガイドしてください。
