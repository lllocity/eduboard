# Step 3.1: doGet関数のグローバルスコープ露出エラーの修正とコミット

## 目的
GASのWebアプリデプロイ時に「スクリプト関数が見つかりません: doGet」というエラーが発生している問題を解消する。バンドラー（esbuild等）によって `doGet` がローカルスコープに隠蔽されていることが原因であるため、明示的にグローバルスコープへ露出させる。問題解消後、修正内容をGitコミットする。

## タスク
1. エントリーポイントとなるファイル（例: `src/main.ts`）を修正すること。
2. `doGet` 関数（および必要であれば `doPost` などのGASのエンドポイント関数）を、以下のようなアプローチで明示的に `global` オブジェクトにアサインする記述に変更すること。

   ```typescript
   // 例: logic.ts などから処理をインポート
   import { handleDoGet } from './logic';

   // GASのグローバルスコープに明示的に登録する
   declare const global: any;
   global.doGet = (e: any) => {
     return handleDoGet(e); // または直接ここにロジックを記述
   };
   ```

3. もし `esbuild-gas-plugin` や `gas-webpack-plugin` などのプラグインを使用している場合は、その設定（`esbuild.js` 等のビルドスクリプト）が正しく機能してグローバル露出が行われているか確認し、機能していなければ上記の手動アサイン方式に切り替えること。
4. 修正後、`npm run build`（バンドル実行）が成功し、出力先のファイル（例: `dist/main.js`）に `global.doGet = ...` のような記述が含まれていること、および `npm test` がPassすることを確認すること。
5. **[追加]** 上記のビルドとテストの成功を確認した後、変更されたファイルをステージング（`git add .`）し、今回の修正内容を簡潔に要約した日本語のコミットメッセージで `git commit` を実行すること。

## 注意事項
- この修正後もローカル環境でのテスト（`npm test`）が引き続きPass状態を維持していることを確認すること。
- コミットメッセージは `fix: doGet関数のグローバルスコープ露出エラーを修正` のような、プレフィックスを用いた分かりやすい形式とすること。
