# Step 4: UIの横型レイアウト化とチェックボックスの拡張

## 目的
月別チェックリストのUIを根本的に見直し、横スクロール型のレイアウトに変更する。また、A4用紙の横向き（ランドスケープ）印刷に最適化し、テスト運用に向けた実用的なチェックボックス（誤答・復習済）を追加する。問題解消後、修正内容をGitコミットする。

## タスク
1. **レイアウトの横型化（フロントエンド改修）:**
   - `index.html`（または対応するVue/Alpineコンポーネント）のレイアウトを変更する。
   - 月ごとのセクションを縦に積むのではなく、横一列に並べる（水平スクロール可能な `flex` コンテナ等を使用）。
   - 各月の列（カラム）の中に、その月に学習する科目と単元が縦にリストアップされるカード型のUI（カンバンボードのような見た目）を構築すること。

2. **チェックボックスとメモ欄の拡張:**
   - 各単元の行に、以下の2つの手書き用チェックボックス（デザイン上の四角 `[ ]`）を並べて配置すること。
     - `[ ] 誤答` （不正解があった場合にチェック）
     - `[ ] 復習済` （復習が完了した場合にチェック）
   - 上記のチェックボックスの横に、手書きで書き込める広めの「メモ・分析用余白」を確保すること。

3. **印刷レイアウトの横向き（ランドスケープ）最適化:**
   - `@media print` 内のCSSを改修し、印刷時の用紙方向を横向きに指定すること（例: `@page { size: A4 landscape; margin: 1cm; }`）。
   - 横長の用紙に対して、月ごとの列が綺麗に収まるように幅を調整すること（必要に応じて、数ヶ月ごとに改ページされるようなCSS制御 `break-inside: avoid` や `page-break-after` などを検討する）。
   - ブラウザ上のスクロールバーや、印刷に不要なボタン類が印刷時に非表示になるよう徹底すること。

4. **テスト・ビルドと自動コミット:**
   - UI修正後、`npm run build` および `npm test` が成功することを確認する。
   - ブラウザでローカルまたはGASのテストデプロイ画面を開き、横スクロールと横向き印刷プレビューが意図通りに機能することを確認する。
   - 確認後、変更されたファイルをステージング（`git add .`）し、`feat: 月別リストの横型レイアウト化とチェックボックスの拡張` という要約で `git commit` を実行すること。

## 注意事項
- デザインには引き続きTailwind CSSを活用し、モダンで視認性の高いUIを保つこと。
- スプレッドシートからのデータ取得ロジック（バックエンド）は今回変更しない。あくまでフロントエンドの描画方式の変更に留めること。
