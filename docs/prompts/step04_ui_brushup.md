# Step 4: UIの月別横スクロール化と手書きレイアウトへの最適化

## 目的
タブUIと年間マップ（ビューA）を廃止し、月別チェックリストを横スクロール型のレイアウトに変更する。また、スプレッドシートのデータから「メモ/備考」を削除しつつ、画面上・印刷時には手書き用の物理的な余白として十分なスペースを確保する。問題解消後、修正内容をGitコミットする。

## タスク
1. **バックエンド・テストのデータ定義修正:**
   - 取得する必須カラムの検証ロジックから「メモ」および「備考」を除外する。
   - `tests/logic.test.ts` などのテストコードも上記に合わせて修正し、`npm test` が通るようにする。

2. **UIの単一化と全体レイアウト:**
   - ビューA（全体俯瞰マトリクス）およびタブ切り替えのUIを削除する。
   - メインコンテナを横スクロール可能（`overflow-x-auto`等）にし、各月（対象月）のセクションを横一列に並べる（`flex`等を使用）。
   - ブラウザ画面に収まりきらなくてもよいため、各月の列は十分に広い固定幅（例: `min-w-[500px]` またはそれ以上）を持たせること。

3. **階層型リストとテキストの折り返し制御:**
   - 月の列の中で、「科目」＞「大項目」＞「単元名」の階層構造でインデントをつけて表示する。
   - 「大項目」および「単元名」のテキストは15文字程度まで折り返されないよう、CSSで制御する（`whitespace-nowrap` や `min-w-[15em]` などを活用）。

4. **チェックボックスと手書きメモ欄の実装:**
   - 単元名と同じ行の右側に配置する要素として、以下のレイアウトを構築する。
     - `[ ] 誤答` （不正解チェック用）
     - `[ ] 復習済` （復習完了チェック用）
     - **メモ欄:** データとしてはバインドしないが、ユーザーが印刷後に手書きでガッツリ書き込めるよう、単元行の右端に十分な空白（`flex-grow`等で最大限広げた余白領域や下線）を明示的に確保する。

5. **自動コミット:**
   - `npm run build` と `npm test` の成功を確認する。
   - 変更されたファイルをステージングし、`feat: UIを月別横スクロールに統一し手書きスペースを確保` という要約で `git commit` を実行する。

## 注意事項
- デザインにはTailwind CSSを使用し、印刷時（`@media print`）にも横並びのレイアウトやチェックボックス、メモ欄の余白が崩れないよう配慮すること。
