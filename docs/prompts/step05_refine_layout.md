# Step 5: 月別リストの階層型レイアウトへの刷新と項目名変更

## 目的
提供された手書きのワイヤーフレームに基づき、月別チェックリスト（横スクロールレイアウト）のUIを階層型（科目 ＞ 大項目 ＞ 単元名）に刷新する。また、データ項目の「備考」を「メモ」に変更し、チェックボックスが整然と並ぶ実用的な印刷用レイアウトを構築する。問題解消後、修正内容をGitコミットする。

## タスク
1. **データ定義の更新 (バックエンド/フロントエンド):**
   - スプレッドシート側のカラム変更に合わせて、取得・表示するデータ項目の `備考` を `メモ` に変更する（TypeScriptのインターフェース定義等も修正すること）。

2. **データの階層化（グループ化）:**
   - フロントエンド（またはバックエンドでのデータ整形時）において、フラットな配列データを以下の階層構造にグループ化するロジックを実装すること。
     - 第1階層: `対象月` (月ごとのカラム)
     - 第2階層: `科目` (例: 算数、理科)
     - 第3階層: `大項目` (例: 総まとめ、外部模試)
     - 第4階層: `単元名` (その大項目に属する単元のリスト)

3. **UIレイアウトの実装 (Tailwind CSS):**
   - **科目ヘッダー:** 四角く囲むなどして目立たせること。
   - **大項目ヘッダー:** 科目の下に配置し、大項目ごとに四角く囲むなどして目立たせること。大項目は同月に複数あるため■などのマーカー付きで表示する。
   - **単元リスト:** 大項目の下にさらにインデントを下げて箇条書き（ドット等のリストマーカー付き）で表示する。
   - **チェックボックスとメモ欄:** 単元名と同じ行の右側に、TailwindのGridレイアウト等を用いて綺麗に縦揃えで配置する。
     - ヘッダー部分（大項目と同じ行など）に「誤答」「復習」「メモ」という見出しをつけること。
     - メモにはプリントしてから手書きで記入できる余白を設けること。
     - 各単元の行には `[ ]` (誤答チェック用)、 `[ ]` (復習済チェック用)、および手書き用の広い空白（メモ欄）を設ける。

4. **印刷対応とコミット:**
   - 変更した階層型レイアウトが、横向き印刷（ランドスケープ）時にもレイアウト崩れを起こさず、手書きスペースが十分に確保されるよう `@media print` を調整すること。
   - 動作確認（`npm run build` および `npm test`）後、変更されたファイルをステージングし、`feat: 月別リストの階層型レイアウトへの刷新とメモ欄への変更` という要約で `git commit` を実行すること。

## 注意事項
- デザインは引き続きモダンでクリーンな状態を保ちつつ、あくまで「印刷して手書きする」際の実用性（余白の広さ、チェックボックスの配置）を最優先すること。
