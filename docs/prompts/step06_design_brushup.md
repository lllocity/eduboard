# Step 6: UIデザインのモダン化と印刷用ディテールの洗練

## 目的
現在の機能とレイアウト構造（横スクロール、階層表示）は完全に維持したまま、Tailwind CSSを用いたスタイリングを全面的に見直し、重く野暮ったいデザインから「モダンでクリーンなWebアプリ」風のデザインへブラッシュアップする。問題解消後、修正内容をGitコミットする。

## タスク（スタイリングの変更）
1. **配色の軽量化（クリーンなUIへ）:**
   - 画面全体のメインヘッダー（EduBoard）の背景色を白（`bg-white`）にし、薄いシャドウ（`shadow-sm`）と下線（`border-b`）で区切る、モダンなSaaS風に変更する。文字色はダークグレーにする。
   - 月別のカラムヘッダー（「4月」「7月」など）の濃い背景色を廃止し、ライトグレー（例: `bg-slate-100` や `bg-gray-50`）にダークグレーの文字色を合わせる。

2. **バッジとタイポグラフィの洗練:**
   - 科目（「国語」「算数」等）のバッジ背景色を、非常に薄い色（例: `bg-indigo-50 text-indigo-700` など）に変更し、角丸（`rounded-md` または `rounded-full`）で柔らかくする。
   - 大項目や単元名のフォントサイズ、ウェイト、文字色（真っ黒ではなく `text-slate-800` など少し落とした色）を調整し、階層がより美しく見えるようマージンを見直す。区切りの罫線は極薄くする（`border-gray-100` など）。

3. **手書き用UIパーツの実装（テキスト記号からの脱却）:**
   - 「誤答」「復習済」の `□` テキストを削除し、Tailwindで本物の四角いボックス（例: `w-5 h-5 border border-gray-400 rounded-sm inline-block mr-1`）をコーディングして配置する。
   - メモ欄の下線を、主張が強すぎない「点線（`border-dotted`）」または「極薄い実線」に変更する。

4. **テスト・ビルドと自動コミット:**
   - UI修正後、`npm run build` が成功することを確認する。
   - 変更されたファイルをステージングし、`refactor: UIデザインをクリーンでモダンな配色・パーツにブラッシュアップ` という要約で `git commit` を実行する。

## 注意事項
- データ構造やロジック、レスポンシブ（横スクロール）の挙動、印刷用設定（`@media print`）は絶対に壊さないこと。見た目の変更のみに集中する。
